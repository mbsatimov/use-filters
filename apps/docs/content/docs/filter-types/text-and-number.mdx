## `f.text`

Free text — search boxes, names, anything typed. Stored and returned as a string.

```ts
search: f.text({ label: 'Search', placeholder: 'Name or SKU' });
// params.search → string | null
```

A search box usually wants a [debounced commit](/docs/concepts/values-and-commits)
so it doesn't refetch on every keystroke:

```ts
search: f.text({ label: 'Search', commit: { debounce: 400 } });
```

## `f.number`

A single number — amounts, counts, rates. Keeps decimals by default; pass
`precision: 'int'` to parse whole numbers only.

```ts
min_price: f.number({ label: 'Min price' });
page_count: f.number({ label: 'Pages', precision: 'int' });
// params.min_price → number | null
```

## `f.numberRange`

A `[min, max]` pair — "price between", "age between". Same `precision` option as
`f.number`; the value is a two-element tuple.

```ts
price: f.numberRange({ label: 'Price' });
// params.price → [number, number] | null
// onChange([min, max])
```

<Callout>
  Need a display hint like a currency symbol? Declare a `unit` field on `NumberFilterMeta` /
  `NumberRangeFilterMeta` and pass it via `meta` — see [UI metadata](/docs/guides/ui-metadata).
  (`f.number`'s own `unit` option is deprecated for the same reason.)
</Callout>
