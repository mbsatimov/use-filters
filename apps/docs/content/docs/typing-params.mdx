---
title: Typing params from your API
description: Validate your filter config against your list endpoint's params type.
---

Pass your list endpoint's params type as `useFilters<ListParams>` to get key
autocomplete, validation that each filter produces the right type, and a `params`
shaped like your API expects:

```ts
interface LoanListParams {
  search?: string;
  status?: 'open' | 'closed';
  customer_id?: number;
  page: number; // pagination keys are owned by the hook — exclude these
  per_page: number; // from what you validate against
}

const { params } = useFilters<LoanListParams>({
  search: f.text({ label: 'Search' }),
  status: f.select({ label: 'Status', options: statusOptions }),
  customer_id: f.asyncSelect({ label: 'Customer', loadOptions })
});
// A key not in LoanListParams, or a filter whose value doesn't fit it, is a
// compile error.
```

<Callout>
  Passing `useFilters<ListParams>` validates against your type; omitting it infers
  `params` per config. TypeScript can't do both at once — pick whichever you need
  for a given screen.
</Callout>
