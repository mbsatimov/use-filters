# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets).

We have a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md).

## This project

Run `npm run changeset` when your change is user-facing (the README/CHANGELOG
audience, not internal refactors/tests/tooling), pick the bump type, and
describe the change the way you'd want it to read in `CHANGELOG.md` — that
description is used close to verbatim. This project follows
[Semantic Versioning](https://semver.org/); while pre-1.0, a `minor` bump may
include breaking changes (there's no `major` bump pre-1.0), so pick `minor`
for anything breaking and `patch` for anything that isn't.

Pushing to `main` with pending changesets opens/updates a "Version Packages"
PR (bumps `package.json`, rewrites `CHANGELOG.md`, consumes the changesets).
Merging that PR publishes to npm automatically — no more manual
`gh release create`.
